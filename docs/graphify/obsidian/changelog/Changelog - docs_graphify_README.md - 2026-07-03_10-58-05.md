+# Obsidian Project Knowledge Base
+
+This project uses a dual-engine automated documentation system combining **Graphify** (for AST and dependency graph extraction) and a custom generator script (for Markdown scaffolding). Together, they produce a complete, searchable, and highly interconnected **Obsidian Vault** that acts as the single source of truth for the architecture.
+
+## 📂 Project Structure
+
+All generated documentation is stored in `docs/graphify/`:
+
+- **`obsidian/`**: The complete Obsidian vault. Open this folder in Obsidian.
+- **`reports/`**: Statistical reports and project metrics (e.g., Validation Report, Dependency Report).
+- **`graphs/`**: Raw `graph.json` files generated by Graphify.
+- **`html/`**: Interactive Callflow HTML diagrams generated by Graphify.
+- **`mermaid/` / `assets/`**: Additional diagrams and media files.
+
+## 🔄 Regeneration Workflow
+
+The documentation is completely dynamic and does not require manual hardcoding of new folders or Python modules. To regenerate the documentation after significant codebase changes, run:
+
+```bat
+.\generate_docs.bat
+```
+
+This script will:
+1. Run `generate_obsidian_kb.py` to auto-discover newly added modules and APIs, generating their respective Markdown files and linking them.
+2. Run `graphify` to rebuild the AST graph.
+3. Export modular HTML visualizations (e.g., separating frontend/backend to avoid node limits).
+4. Perform validation (checking for broken links and orphaned notes).
+
+## 📖 Using Obsidian & Mermaid
+
+1. Open **Obsidian** and select **"Open folder as vault"**. Choose the `docs/graphify/obsidian` folder.
+2. Use the **Graph View** in Obsidian to visualize the connections between services, APIs, and pipelines.
+3. Obsidian natively supports Mermaid diagrams. You will see dynamically generated architecture diagrams rendered directly in the Markdown notes.
+
+## ➕ Adding New Modules
+
+You do not need to manually create new documentation pages. When you add a new service folder (e.g., `new-pipeline/`), simply run `generate_docs.bat`. The generator will detect the directory, create `Module New-pipeline.md`, and automatically link it to the project's dependencies!
+
+## ⬆️ Update Workflow & Troubleshooting
+
+- **Graphify Update:** To update Graphify, run `.\.venv\Scripts\python.exe -m pip install --upgrade graphifyy`.
+- **Missing Diagrams/Graphs:** If `callflow.html` fails, it may be due to graph size limits. Check `docs/graphify/html/backend_callflow.html` for modular diagrams.
+- **Validation Errors:** Check `Validation Report.md` inside Obsidian. If it lists broken links, it means a referenced service might have been deleted from the repository.  # Truncated to 5000 chars for readability if huge
